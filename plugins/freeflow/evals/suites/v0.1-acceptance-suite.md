## Purpose

Define the small local-only gate for calling the current Freeflow plugin draft v0.1-ready for Hassan's Codex-first dogfooding.

This suite is not a public release gate and not a hook/CLI requirement. It is a curated set of existing evals that covers the core product promise without rerunning every historical regression.

## Pass Standard

The suite passes only when:

- every required fixture eval scores all assertions as passing from final response plus diff evidence
- no required eval edits source truth to erase a contradiction
- no required eval creates unnecessary workflow artifacts for small reversible work
- no required eval commits, stages, or mutates git state outside the fixture expectation
- `audit-command-surface.sh` passes
- no run leaves saved output files in the repo root

For fixture evals, diff evidence beats final-response claims.

## Commands

Use a unique run folder per eval. Always pass a full output path under `plugins/freeflow/evals/runs/`:

```sh
plugins/freeflow/evals/scripts/run-fixture-eval.sh \
  STP-001 --with-skill \
  --run-dir plugins/freeflow/evals/runs/v0.1-acceptance/stp-001 \
  --output plugins/freeflow/evals/runs/v0.1-acceptance/stp-001-with-skill-output.md \
  plugins/freeflow/skills/setup-freeflow/SKILL.md
```

Before spending model tokens, dry-run each command:

```sh
plugins/freeflow/evals/scripts/run-fixture-eval.sh STP-001 --dry-run
```

For always-on installed-memory coverage:

```sh
plugins/freeflow/evals/scripts/run-fixture-eval.sh \
  AON-001 --with-skill \
  --run-dir plugins/freeflow/evals/runs/v0.1-acceptance/aon-001 \
  --output plugins/freeflow/evals/runs/v0.1-acceptance/aon-001-with-skill-output.md
```

For command-surface consistency:

```sh
plugins/freeflow/evals/scripts/audit-command-surface.sh
```

## Reporting

Create `plugins/freeflow/evals/reports/acceptance/v0.1-acceptance-report.md` after running the suite.

The report should include:

- date, model, runner, and harness command shape
- pass/fail table for each required eval
- links to saved outputs and diffs
- scoring table or explicit assertion summary
- root-output check result
- `audit-command-surface.sh` result
- any known non-blocking smoke-test gaps

Do not mark v0.1 accepted from old reports alone. Old reports can explain why an eval is in the suite, but the suite result needs fresh saved runs.

## Deferred Coverage

Not required for local v0.1:

- live Claude paired smoke evals
- native slash handlers
- hooks or CLI enforcement
- every historical `CMD-*` command eval
- optional `handoff/references/templates.md`

Add these only when local dogfooding or targeted evals show a concrete gap.
